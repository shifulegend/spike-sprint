# Documentation Rules
<!-- DYNAMIC FILE -->
<!-- paths: **/*.md, docs/** -->

- Timestamp every documentation update.
- Keep docs factual, concise, and repo-specific.
- Update `docs/ai/mistakes.md` immediately when a mistake is found.
- Update `docs/ai/decision-log.md` when a durable decision is made.
- Update `docs/ai/change-trace.md` after each meaningful sub-step.
- Newest entries first in all log files.
- Never defer documentation — write it as soon as the information is known.
